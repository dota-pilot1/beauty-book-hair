## 2. 지금까지 한 작업

### 예약 CRUD 전체 구현 (Phase 3~9)
- `POST /api/reservations` 예약 생성 (슬롯 충돌 검증 포함)
- `GET /api/reservations/me` 내 예약 조회
- `GET /api/reservations?date=` 관리자 날짜별 조회 (ADMIN/MANAGER 전용)
- `PATCH /api/reservations/{id}/status` 상태 변경 + adminMemo(취소 사유) 저장
- `ChangeReservationStatusRequest`에 `adminMemo` 추가, `Reservation.changeStatus(status, adminMemo)` 시그니처 변경

### 고객 예약 화면 완성
- 시술→디자이너→날짜/시간 3단계 플로우 (bookingFlowStore localStorage 영속)
- 예약 요청 후 `/reservations`로 이동

### 예약 현황 페이지 (/reservations)
- 날짜 탭(7일) + 예약 목록
- 관리자: 전체/나 토글, 5버튼 상태 변경(승인대기·확정·완료·노쇼·취소), 취소 사유 입력
- 고객: 본인 예약 + 취소 버튼

### 대시보드 리디자인
- AdminShell 컴포넌트 신설 (CustomerShell과 동일한 디자인 시스템)
- 오늘 예약 현황 실 데이터 연결, 통계 카드(전체·승인대기·확정)

### 내 예약 (/my-reservations)
- 실 데이터 연결, 진행 중/지난 예약 분리, 고객 취소 연결

### 이전 작업
- Designer → Staff 전면 리네임
- 직원 관리 화면 (/staff)
- 시술 시드 27개 확충
