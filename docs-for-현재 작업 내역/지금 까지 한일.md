## 운영 장애 수정 (이번 세션)

### 1. 일요일 "휴무일입니다" 오표시
- 운영 DB `business_hours` 테이블에 SUNDAY `closed = true`, `open_time = null` 로 초기화되어 있었음
- SSH → docker exec로 직접 UPDATE 수정

### 2. /reservations 페이지 Application error
- 원인: `pendingQuery.data ?? []` — 서버 에러 응답이 non-array일 때 `.map()` 실패
- 수정: `Array.isArray()` 방어 처리로 전면 교체

### 3. 운영 로그인 불가 (CORS → Mixed Content)
- 1차: `.env.production` 없이 빌드 → `localhost:4101` 하드코딩 → CORS 에러
- 2차: EC2 IP 직접 호출 → HTTPS 페이지에서 HTTP 차단 (Mixed Content)
- 최종 수정: `.env.production` API URL을 `https://dxline-tallent.com` 으로 변경 → CloudFront `/api/*` 프록시 경유

**관련 파일**
- `beauty-book--front/.env.production` — `NEXT_PUBLIC_API_BASE_URL=https://dxline-tallent.com`
- `개발 관리/디버깅 히스토리/2026-04-25 운영 로그인 실패 (CORS + CloudFront 함정).md`
- `개발 관리/디버깅 히스토리/2026-04-26 운영 Mixed Content (백엔드 직접 호출 차단).md`

### 4. 운영 예약 요청 응답 없음 (ThrowableProxy)
- 원인: Tomcat 클래스로더 ↔ Spring Boot 중첩 클래스로더 분리 문제
- 수정: `application.yaml` Tomcat 로거 OFF

**관련 파일**
- `beauty-book-server/src/main/resources/application.yaml`
- `개발 관리/디버깅 히스토리/2026-04-26 운영 예약 요청 응답 없음 (logback ThrowableProxy).md`
