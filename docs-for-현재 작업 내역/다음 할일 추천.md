## 🔜 다음 할일

### 1순위: 관리자 스케쥴 관리 화면

**왜 먼저 해야 하는가**
지금은 영업시간·근무시간·예약불가시간이 모두 시드 데이터에 하드코딩되어 있어 실제 운영이 불가능하다.
API는 Phase 2에서 이미 전부 구현되어 있으므로 UI만 붙이면 된다.

**할 일**
- 영업시간 편집 UI: 요일별 오픈/마감 시간, 휴무일 설정
- 디자이너별 근무시간 편집 UI: 요일별 출근/퇴근, 미근무 설정
- 예약 불가 시간(BlockedTime) 목록 + 생성/삭제 UI

**관련 API (이미 구현됨)**
- `GET/PUT /api/admin/schedule/business-hours`
- `GET/PUT /api/admin/schedule/staff/{staffId}/working-hours`
- `GET/POST/DELETE /api/admin/schedule/blocked-times`

### 2순위: 고객 알림 (예약 확정/취소 시)

**왜 필요한가**
관리자가 예약을 승인해도 고객이 아무 통보를 받지 못하는 것이 현재 가장 큰 UX 구멍이다.

**접근 방법**
- 단기: JavaMailSender로 이메일 알림 (심사 불필요, 빠르게 붙일 수 있음)
- 장기: 카카오 알림톡 연동 (심사 필요, 실제 미용실 운영 표준)

**서버 작업**
- `ReservationService.changeStatus()` 내 상태 변경 후 알림 이벤트 발행
- `ApplicationEventPublisher` + `@EventListener` 패턴 권장 (서비스 결합도 최소화)
