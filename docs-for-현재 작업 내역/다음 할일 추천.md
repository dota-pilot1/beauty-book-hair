## 1순위: 고객 알림 (예약 확정/취소 시)

**왜 필요한가**
관리자가 예약을 승인해도 고객이 아무 통보를 받지 못함. 가장 큰 UX 구멍.

**접근 방법**
- 단기: JavaMailSender 이메일 알림 (심사 불필요, 빠르게 붙일 수 있음)
- 장기: 카카오 알림톡 연동 (심사 필요, 실제 미용실 운영 표준)

**서버 작업 포인트**
- `ReservationService.changeStatus()` 내 상태 변경 후 이벤트 발행
- `ApplicationEventPublisher` + `@EventListener` 패턴 권장 (서비스 결합도 최소화)

**관련 파일**
- `beauty-book-server/src/main/java/…/reservation/ReservationService.java`

## 3순위: AdminShell ↔ AdminSidebar 라우트 정리

**왜 필요한가**
- `/dashboard`, `/reservations` 등 비-admin 경로와 `/admin/dashboard`, `/admin/reservations` 경로가 이중으로 존재
- 사이드바도 `AdminSidebar`와 `AdminShell` 내부 사이드바로 이중 관리 중

**접근 방법**
- 모든 관리자 페이지를 `/admin/*` 하위로 통일
- `AdminShell`에서 사이드바 제거 → `AdminSidebar` 단일 사용
- 기존 `/dashboard`, `/reservations` 등에 redirect 처리

**관련 파일**
- `beauty-book--front/src/shared/ui/admin/AdminShell.tsx`
- `beauty-book--front/src/shared/ui/admin/AdminSidebar.tsx`
- `beauty-book--front/src/app/dashboard/page.tsx`
- `beauty-book--front/src/app/admin/dashboard/page.tsx`

## 참고 — 운영 배포 체크리스트

```
□ .env.production 확인 (NEXT_PUBLIC_API_BASE_URL=https://dxline-tallent.com)
□ npm run build
□ aws s3 sync out/ s3://beauty-book-hair-front --delete --exclude "beauty-book/*"   ← ⚠️ exclude 필수
□ aws cloudfront create-invalidation --distribution-id E11NF3HMOB52NI --paths "/*"
□ /schedule 접속 → 요일별 영업시간 확인 (특히 SUNDAY)
□ /staff 접속 → 직원별 SUNDAY 근무 여부 확인
□ /booking 접속 → 오늘 날짜 슬롯 정상 노출 확인
□ /customer-space 접속 → 이번 주 스케쥴 카드 정상 표시 확인
```

**관련 파일**
- `CLAUDE.md` — 전체 배포 가이드
- `docs-for-운영 정보 관리/구현 현황 및 할일.md` — 운영 이슈 및 TODO 전체 목록
- `개발 관리/디버깅 히스토리/` — 장애 대응 이력
