## 1순위: 게시판 관련 후속 작업

### 1-A. 게시글 상세 페이지 (`/boards/[code]/[id]`) 정비
- 현재 SSG `generateStaticParams`가 `__placeholder__/0`만 생성 → 직접 URL 접속 시 404
- `generateStaticParams`에서 빌드 시 게시글 목록도 fetch해 정적 페이지 생성 (또는 ISR 검토)
- OG 메타태그 추가 (소셜 공유 대비)

### 1-B. 게시판 종류별 UI 분기
- `BoardKind`: NOTICE / COMMUNITY / QA / FAQ / GALLERY
- GALLERY: 이미지 그리드 레이아웃, QA: 답변 완료 뱃지 강조 등 kind별 뷰 분기
- FAQ: 아코디언 형태 고려

### 1-C. 게시판 추가 시 재빌드 자동화 안내
- 현재 게시판 코드를 추가하면 재빌드·재배포 해야 `generateStaticParams`에 반영됨
- 팀에 공지 또는 README에 명시

## 2순위: 고객 알림 (예약 확정/취소 시)

**왜 필요한가**
관리자가 예약을 승인해도 고객이 아무 통보를 받지 못함. 가장 큰 UX 구멍.

**접근 방법**
- 단기: JavaMailSender 이메일 알림 (심사 불필요, 빠르게 붙일 수 있음)
- 장기: 카카오 알림톡 연동 (심사 필요, 실제 미용실 운영 표준)

**서버 작업 포인트**
- `ReservationService.changeStatus()` 내 상태 변경 후 이벤트 발행
- `ApplicationEventPublisher` + `@EventListener` 패턴 권장

**관련 파일**
- `beauty-book-server/src/main/java/…/reservation/ReservationService.java`

## 3순위: 게시판 검색 기능

**왜 필요한가**
게시글이 쌓이면 목록 스크롤로는 찾기 어려움. 특히 공지·FAQ 성격의 게시판에서 필수.

**접근 방법**
- 백엔드: `GET /api/boards/{code}?q=검색어` 파라미터 추가, JPA `LIKE` 또는 `Contains` 쿼리
- 프론트: 목록 상단에 검색 input, `useDebounce` 후 쿼리 전달

## 참고 — 운영 배포 체크리스트

```
□ .env.production 확인 (NEXT_PUBLIC_API_BASE_URL=https://dxline-tallent.com)
□ npm run build  ← 빌드 시 /api/boards/configs 자동 호출해 게시판 페이지 생성
□ aws s3 sync out/ s3://beauty-book-hair-front --delete --exclude "beauty-book/*"   ← ⚠️ exclude 필수
□ aws cloudfront create-invalidation --distribution-id E11NF3HMOB52NI --paths "/*"
□ /schedule 접속 → 요일별 영업시간 확인 (특히 SUNDAY)
□ /staff 접속 → 직원별 SUNDAY 근무 여부 확인
□ /booking 접속 → 오늘 날짜 슬롯 정상 노출 확인
□ /boards/notice 접속 → 게시판 목록·상세·댓글 정상 동작 확인
□ /customer-space 접속 → 이번 주 스케쥴 카드 정상 표시 확인
```

**관련 파일**
- `CLAUDE.md` — 전체 배포 가이드
- `개발 관리/디버깅 히스토리/` — 장애 대응 이력
