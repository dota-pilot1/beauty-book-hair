## theme.ts 토큰 변환 메모

towercrane은 자체 디자인 토큰(`text-text-primary`, `bg-surface-muted` 등)을 사용.
beauty-book은 Tailwind + shadcn 기반이므로 아래처럼 교체.

| towercrane 토큰 | beauty-book 대체 |
|----------------|-----------------|
| `text-text-primary` | `text-foreground` |
| `text-text-secondary` | `text-muted-foreground` |
| `text-text-muted` | `text-muted-foreground/60` |
| `bg-surface-muted` | `bg-muted` |
| `bg-surface-strong` | `bg-accent` |
| `border-surface-border` | `border-border` |
| `text-brand-primary` | `text-primary` |
| `bg-brand-glass` | `bg-primary/10` |

## 게시판 적용 포인트

### 게시글 작성 폼
```tsx
// src/features/board/BoardPostEditor.tsx
<LexicalEditor
  namespace="BoardEditor"
  initialState={post?.content}   // DB에 저장된 Lexical JSON
  onChange={(json) => setValue('content', json)}
  placeholder="내용을 입력하세요..."
  minHeight="300px"
/>
```

### 게시글 뷰어 (readOnly)
```tsx
<LexicalEditor
  namespace="BoardViewer"
  initialState={post.content}
  onChange={() => {}}
  readOnly={true}
/>
```

### DB 저장 형식
- `boards.content` 컬럼: `TEXT` 타입, Lexical EditorState JSON 문자열 저장
- Phase 1에서 textarea로 작성한 plain text는 InitialContentPlugin이 자동 fallback 처리

---

## InitialContentPlugin 동작 원리

towercrane 구현체에 이미 포함됨. plain text/JSON 혼용 데이터 자동 처리:

```
1. JSON.parse 시도
2. parsed.root 있음 → Lexical EditorState로 로드
3. 파싱 실패 → 빈 에디터로 시작 (Phase 1 textarea 데이터는 별도 마이그레이션 필요)
```

Phase 1 → Phase 2 전환 시 plain text를 paragraph 노드로 변환하는 마이그레이션 스크립트 작성 필요.

## S3 이미지 업로드 연동

beauty-book 백엔드에 presigned URL API가 없으면 Phase 2 이미지 기능 전에 추가 필요.

```http
POST /api/upload/presign
Body: { filename, contentType }
Response: { presignedUrl, publicUrl, key }
```

기존 `SiteSettingsForm`의 이미지 업로드 로직 참고.
