# 헤어 다이어리(블로그) 구현 계획

> 작성일: 2026-04-27

---

## 왜 Board 시스템 재활용이 아닌 별도 구현인가?

| 항목 | 게시판 (Board) | 블로그 (Blog) |
|------|---------------|--------------|
| URL | `/boards/notice/123` — id 기반 | `/blog/예쁜-웨이브-스타일` — **slug** 기반 |
| SEO | 없음 | slug, og:image, meta description **필수** |
| 대표 이미지 | 없음 | 커버 이미지가 핵심 구성 요소 |
| 작성자 | 이름 문자열 | Staff 엔티티와 연결 (프로필 카드) |
| 콘텐츠 발견 | 목록 나열 | 태그 페이지, 관련 포스트 |
| 발행 흐름 | 작성 → 게시 | **DRAFT → PUBLISHED** 상태 전환 |

→ **별도 `blog` 모듈로 구현한다.**

### `BlogPost.java` — 핵심 필드

```java
@Entity
@SQLRestriction("deleted_at IS NULL")
@Table(name = "blog_posts")
public class BlogPost {
    Long id;
    String slug;              // unique, URL 식별자
    String title;
    String content;           // Lexical JSON (TEXT)
    String summary;           // 카드 요약 / og:description
    String coverImageUrl;

    // Staff(=Designer) — Gallery 패턴처럼 denormalize (FK 미사용)
    Long authorStaffId;
    String authorName;

    BlogPostStatus status = BlogPostStatus.DRAFT;
    int viewCount = 0;
    boolean isPinned = false;

    @ManyToMany
    @JoinTable(name = "blog_post_tags", ...)
    Set<BlogTag> tags;

    Instant publishedAt;
    Instant createdAt;
    Instant updatedAt;
    Instant deletedAt;

    // 정적 팩토리 + 행위 메서드 (Board 패턴 동일)
    public static BlogPost create(...) { ... }
    public void update(...) { ... }
    public void publish() { status = PUBLISHED; if (publishedAt == null) publishedAt = Instant.now(); }
    public void setTags(Set<BlogTag> tags) { ... }
    public void incrementView() { viewCount++; }
    public void softDelete() { deletedAt = Instant.now(); }
}
```

### `BlogPostRepository.java`

```java
public interface BlogPostRepository extends JpaRepository<BlogPost, Long> {
    Page<BlogPost> findByStatus(BlogPostStatus status, Pageable pageable);

    @Query("SELECT DISTINCT p FROM BlogPost p JOIN p.tags t " +
           "WHERE p.status = :status AND t.slug = :tagSlug")
    Page<BlogPost> findByStatusAndTagSlug(...);

    @EntityGraph(attributePaths = {"tags"})
    Optional<BlogPost> findWithTagsBySlug(String slug);   // 상세 — N+1 방지

    boolean existsBySlug(String slug);
}
```

### API 엔드포인트

| 메서드 | 경로 | 설명 |
|--------|------|------|
| GET | `/api/blog/posts` | 목록 (태그 필터, 페이지네이션) |
| GET | `/api/blog/posts/{slug}` | 상세 + viewCount 증가 |
| GET | `/api/blog/tags` | 태그 목록 |
| GET | `/api/admin/blog/posts/suggest-slug` | slug 자동 생성 |
| POST | `/api/admin/blog/posts` | 작성 |
| PATCH | `/api/admin/blog/posts/{id}` | 수정 |
| DELETE | `/api/admin/blog/posts/{id}` | 삭제 |
| POST | `/api/admin/blog/tags` | 태그 생성 |
| DELETE | `/api/admin/blog/tags/{id}` | 태그 삭제 |

### Slug 생성 전략

- 어드민 에디터에서 제목 입력 후 "slug 자동 생성" 버튼 → `/api/admin/blog/posts/suggest-slug?title=...`
- 한글 제목은 `{uuid 앞 8자리}` suffix로 중복 방지. 예: `단발-시술-a1b2c3d4`
- DB에 `UNIQUE` 제약 필수


### `/blog` 목록 페이지

- `page.tsx` → Server Component 껍데기 (metadata만)
- `_BlogListClient.tsx` → `"use client"`, 카드 그리드 + 태그 필터 + 페이지네이션
- 카드: 커버 이미지(aspect-video) + isPinned 뱃지 + 제목 + 요약 + 작성자 + 날짜
- 태그 필터 UI는 갤러리 카테고리 패턴 그대로 재사용

### `/blog/[slug]` 상세 페이지

- `page.tsx` → `generateStaticParams` (빌드 시 PUBLISHED slug pre-render) + `generateMetadata` (og:title, og:image, og:description)
- `_BlogPostClient.tsx` → `"use client"`, 커버 이미지 + 메타 + 태그 뱃지 + `LexicalEditor readOnly`
- 태그 클릭 → `/blog?tag={slug}` 이동

### 어드민 페이지

- `_BlogTagManagementSection.tsx`: 태그 목록 + name/slug 추가 폼
- `_BlogPostManagementSection.tsx`: 포스트 테이블 + 상태 표시 + 편집/삭제 액션
- `/blog-management/new`: Lexical 에디터 + 메타 입력 폼 (제목, slug, summary, 커버이미지, 태그 선택, DRAFT/PUBLISHED)

## 주의사항

- **N+1 방지**: 목록 응답에 `tags` 미포함. 상세 조회만 `@EntityGraph(attributePaths = {"tags"})` 적용
- **SSG**: `/blog/[slug]`에서 `generateStaticParams` + `next: { revalidate: 60 }` ISR 적용
- **Staff 연결**: `ManyToOne` FK 대신 `authorStaffId` / `authorName` denormalize — Gallery 패턴과 동일
- **DB 인덱스**: `blog_posts(slug)` UNIQUE, `blog_posts(status, published_at)`, `blog_tags(slug)` UNIQUE
- **ddl-auto: update** 이므로 배포 시 테이블 자동 생성됨 — 별도 마이그레이션 스크립트 불필요
