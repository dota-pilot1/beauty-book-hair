# BeautyBook 보일러플레이트 강화 방법

> 현재 상태: JWT 인증 / 역할·권한 CRUD / 테마 6색 / 다국어 4개국 / 로그인·회원가입 UX 개선 완료
> 다음 단계로 "신뢰 가능한 보일러플레이트"가 되기 위한 핵심 5가지 + 보너스

---

## 우선순위

```
1 (필수)  →  3 (보안)  →  2 (UX)  →  5 (신뢰)  →  4 (성숙도)
```

실서비스 안전성·필수 UX 부터 채우는 순서로 권장.

---

## 1. 이메일 인증 + 비밀번호 재설정 플로우

**Why**
현재 가입만 되고 이메일 검증 / 재설정 플로우가 없음. 가장 자주 발견되는 공백이며 실서비스 투입 전 필수.

**What**
- 가입 직후 인증 메일 발송 → `/verify-email?token=...` 로 검증
- `/forgot-password` → 토큰 발급 → 메일 링크 → `/reset-password?token=...`
- 토큰: 서버에서 `password_reset_token` / `email_verification_token` 테이블 (UUID + expiresAt + used 플래그)
- SMTP 어댑터 (Resend / Mailgun / SES — 인터페이스로 추상화)
- 프론트: 전용 폼 2개 + 성공/만료/재발송 상태 UI

**Impact**
★★★★★ — 없으면 실서비스 불가.

---

## 2. OAuth 소셜 로그인 (Google / GitHub / Kakao)

**Why**
2026 년 기준 소셜 로그인 없는 로그인 페이지는 구식으로 보임. 국내 타겟이면 Kakao, 글로벌이면 Google 이 실질 필수.

**What**
- Spring Security OAuth2 Client 로 3개 프로바이더 설정 (`application.yml`)
- Next.js `/oauth/callback/:provider` 라우트에서 토큰 교환 → 기존 JWT 로 매핑
- `user_oauth_provider` 테이블 (userId, provider, providerUserId)
- 계정 연결·해제 UI (마이페이지)
- 최초 OAuth 가입자도 `ROLE_USER` 기본 부여

**Impact**
★★★★☆ — 신규 가입 전환율 크게 상승.

---

## 3. Refresh Token 회전 + httpOnly 쿠키 저장

**Why**
현재 JWT 가 `localStorage` 에 저장되어 있다면 XSS 취약. 보안 레벨을 한 단계 올려야 "프로덕션급 보일러플레이트"로 불릴 수 있음.

**What**
- Access Token (짧게, 15 분) + Refresh Token (길게, 14 일) 분리
- Refresh Token 은 httpOnly · Secure · SameSite=Lax 쿠키에만 저장
- 프론트: Axios interceptor 로 401 발생 시 silent refresh → 재시도
- 로그아웃: 서버가 Refresh Token 무효화 (블랙리스트 or DB 삭제)
- 토큰 회전 (rotation): refresh 할 때마다 새 refresh token 발급

**Impact**
★★★★★ — 보안 관점에서 가장 큰 업그레이드.

## 4. 2FA (TOTP) + 감사 로그 (Audit Log)

### 4-1. 2FA (TOTP)
- Google Authenticator / 1Password 호환 TOTP
- 서버: `samstevens/totp` 등 라이브러리 활용
- 유저별 secret 발급 → QR 코드 → 6자리 검증
- 관리자 계정에 강제 (설정 토글) — 엔터프라이즈 느낌

### 4-2. Audit Log
- `audit_log` 테이블: `(id, actor_user_id, action, target_type, target_id, payload_json, created_at)`
- 이벤트 예: `ROLE_CREATED`, `PERMISSION_GRANTED`, `USER_ROLE_CHANGED`, `LOGIN_SUCCESS`, `LOGIN_FAILED`
- AOP 로 Role/Permission CRUD 자동 기록
- `/admin/audit-log` 페이지 — 필터(액터·액션·기간) + 페이지네이션

**Why 둘 다**
권한 관리(RBAC) 보일러플레이트에서 **누가 언제 무엇을 바꿨는지** 추적 못 하면 반쪽. 2FA 는 관리자 계정 탈취 방어의 표준.

**Impact**
★★★★☆ — RBAC 가 메인 셀링포인트라면 필수.

## 5. E2E 테스트 + CI 파이프라인

**Why**
"테스트가 있는 보일러플레이트"는 신뢰도가 다름. README 배지 붙이는 순간 인상이 달라짐.

**What**
- **Playwright E2E** (프론트 레포):
  - 로그인 / 가입 / 잘못된 비번 / 비밀번호 토글
  - 권한 없는 유저가 `/users` 접근 → 리다이렉트
  - 관리자가 역할 변경 → 일반 유저 UI 에 반영
- **JUnit 통합 테스트** (서버):
  - `@SpringBootTest` + Testcontainers (PostgreSQL)
  - Role CRUD / Permission 매핑 / JWT 발급·검증
- **GitHub Actions**:
  - Job 1: Frontend — `npm install → typecheck → lint → build → playwright`
  - Job 2: Backend — `./gradlew test → build`
  - Job 3: Docker — 이미지 빌드 캐싱 (PR 에는 빌드만, main 에는 push)
- README 상단 배지: `build | tests | coverage`

**Impact**
★★★★☆ — 보일러플레이트 신뢰도 직결.

## 보너스 후보 (여력 되면)

| 항목 | 이유 |
| --- | --- |
| **계정 설정 페이지** (비밀번호 변경 · 탈퇴 · 프로필) | 현재 `/profile` 은 빈 스텁 — 실제 사용자 관점 UX 완성도 |
| **Rate Limiting + 계정 잠금** | `/login` brute-force 방지 (Bucket4j or Redis 카운터). 5회 실패 시 15분 잠금 |
| **프론트 Dockerfile + 풀스택 docker-compose** | 현재 DB 만 컴포즈 — 한 줄로 전체 스택 띄울 수 있어야 진짜 보일러플레이트 |
| **OpenAPI → TypeScript 클라이언트 자동 생성** | Swagger 가 있으니 `openapi-typescript-codegen` 으로 타입 안전한 API 클라이언트 자동 생성 |
| **에러 모니터링 훅** (Sentry 인터페이스) | 실제 DSN 없이도 훅 포인트 제공 — 사용자가 DSN 만 꽂으면 됨 |
| **테이블·페이지네이션 공통 컴포넌트** | shadcn Table + TanStack Table 래퍼. Users/Roles 페이지 모두에 적용 |
| **에러 바운더리 + 404/500 페이지** | Next.js `not-found.tsx` / `error.tsx` 커스텀 |
| **접근성 감사** (A11y) | 키보드 네비, focus trap (다이얼로그), aria-label 점검 |

## 로드맵 제안

### Phase 1 — 실서비스 가능 최소선 (2~3주)
- [ ] 1번 이메일 인증 + 비밀번호 재설정
- [ ] 3번 Refresh Token + httpOnly 쿠키
- [ ] 보너스: 계정 설정 페이지 + Rate Limiting

### Phase 2 — 차별화 (2~3주)
- [ ] 2번 OAuth (최소 Google + Kakao)
- [ ] 5번 E2E + CI
- [ ] 보너스: 프론트 Dockerfile + 풀스택 컴포즈

### Phase 3 — 엔터프라이즈 완성도 (3~4주)
- [ ] 4번 2FA + Audit Log
- [ ] 보너스: OpenAPI → TS 클라이언트
- [ ] 보너스: 에러 모니터링 훅 + 에러 바운더리

## 작성자 메모

- 위 5개가 갖춰지면 "개인 프로젝트 스타터" 수준에서 "팀이 실제로 쓰는 프로덕션 보일러플레이트" 수준으로 전환됨
- 특히 **1 + 3** 은 안 하면 보안 감사 / 실서비스 투입 불가 — 최우선
- **4 (Audit Log)** 는 RBAC 을 메인 셀링포인트로 가져가려면 필수
- 전체를 한 번에 할 필요 없음. Phase 1 만 해도 가치가 크게 상승
