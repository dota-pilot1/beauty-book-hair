# 메뉴 추가 및 페이지 추가 방법

## 1. 도커 / DB 정보

현재 프로젝트 DB는 [docker-compose.yml](/Users/terecal/beauty-book-hair/docker-compose.yml) 기준으로 PostgreSQL 하나를 사용합니다.

- 컨테이너명: `beauty-book-postgres`
- DB: `beauty_book`
- 포트: `5434`
- 계정: `postgres`
- 비밀번호: `postgres`

백엔드 설정은 [application.yaml](/Users/terecal/beauty-book-hair/beauty-book-server/src/main/resources/application.yaml) 에 있습니다.

- JDBC URL: `jdbc:postgresql://localhost:5434/beauty_book`
- JPA: `ddl-auto: update`

자주 쓰는 명령어:

```bash
docker compose up -d
docker compose ps
docker exec -it beauty-book-postgres psql -U postgres -d beauty_book
```

## 2. 메뉴 추가 방법

현재 상단 메뉴 데이터는 백엔드 시드 코드에서 시작합니다.

- 시드 위치: [MenuSeeder.java](/Users/terecal/beauty-book-hair/beauty-book-server/src/main/java/com/cj/beautybook/config/MenuSeeder.java)
- 메뉴 조회 API: `/api/menus`
- 프론트 메뉴 렌더링: [Header.tsx](/Users/terecal/beauty-book-hair/beauty-book--front/src/widgets/header/ui/Header.tsx)

추가 순서:

1. [MenuSeeder.java](/Users/terecal/beauty-book-hair/beauty-book-server/src/main/java/com/cj/beautybook/config/MenuSeeder.java) 의 `defs` 목록에 메뉴를 추가합니다.
2. 필요한 경우 상위 메뉴 `code` 를 `parentCode` 로 연결합니다.
3. `path` 는 프론트 페이지 경로와 동일하게 맞춥니다.
4. `requiredRole` 을 지정하면 해당 역할만 메뉴를 봅니다.
5. 백엔드를 재시작해서 시드가 반영되게 합니다.

예시:

```java
new MenuDef(
    "ADMIN_BEAUTY_SERVICES",
    "ADMIN",
    "시술 관리",
    null,
    "/beauty-services",
    "Scissors",
    RoleSeeder.ROLE_ADMIN,
    2
)
```

주의:

- 현재 시더는 `code` 가 이미 있으면 건너뜁니다.
- 즉 메뉴 정의를 코드에서 바꿔도 DB에 기존 데이터가 있으면 자동 수정되지 않을 수 있습니다.
- 이 경우 백엔드 재시작만으로 안 바뀌면 `menus` 테이블 값을 직접 수정하거나 시더를 업데이트형으로 바꿔야 합니다.

메뉴 반영 확인:

```bash
curl -s http://localhost:4101/api/menus
```

## 3. 페이지 추가 방법

프론트는 Next.js App Router 구조입니다.

- 페이지 위치: `beauty-book--front/src/app/.../page.tsx`
- 예시 페이지: [beauty-services/page.tsx](/Users/terecal/beauty-book-hair/beauty-book--front/src/app/beauty-services/page.tsx)

추가 순서:

1. `src/app/원하는경로/page.tsx` 생성
2. 필요한 기능 컴포넌트는 `features` 또는 `entities` 아래에 추가
3. 메뉴의 `path` 와 페이지 경로를 동일하게 맞춤
4. 관리자 전용이면 `RequireRole` 같은 가드 적용

예시:

```tsx
"use client";

import { RequireRole } from "@/widgets/guards/RequireRole";

export default function ExamplePage() {
  return (
    <RequireRole roles={["ROLE_ADMIN"]}>
      <main className="w-full px-4 py-4">
        <h1 className="text-2xl font-bold">예시 페이지</h1>
      </main>
    </RequireRole>
  );
}
```

## 4. 메뉴 + 페이지 같이 추가할 때 최소 체크리스트

1. 백엔드 메뉴 시드 추가
2. 프론트 `page.tsx` 추가
3. 필요한 API/컴포넌트 연결
4. 백엔드 재시작
5. `/api/menus` 응답 확인
6. 브라우저 새로고침 후 상단 메뉴 노출 확인

- 고객 허브 기본 홈: `/customer-space`
- 예약 페이지: `/booking`
- 관리자 시술 관리 페이지: `/beauty-services`
- 상단 메뉴는 DB 기반이라 코드만 추가하고 서버를 안 다시 띄우면 화면에 안 보일 수 있습니다.
