## 우선순위 높은 할일

- 예약 상태값 정의

- 권한 정책 초안 반영

### 3. 예약에 필요한 선행 마스터 데이터 만들기

- 고객 관리
- 시술 서비스 관리
- 영업시간/휴무일/직원별 스케줄 정책

## 추천 작업 순서

1. 예약 MVP 데이터 모델 문서화
2. 백엔드 예약 API 골격 구현
3. 예약 관리용 관리자 화면 추가
4. 예약 가능 시간 계산 로직 연결
5. 고객 예약 플로우 확장

## 주의할 점

- 예약 기능은 인증/권한 기반은 준비됐지만, 예약 가능 시간 계산 기준이 아직 없음.
- 그래서 직원 스케줄, 영업시간, 시술 소요시간 중 무엇을 1차에 포함할지 먼저 정하지 않으면 구현이 흔들릴 가능성이 큼.
- 현재 대시보드가 비어 있으므로 예약 현황 위젯을 붙이는 방향으로 자연스럽게 확장 가능.

- 직원/고객/서비스 중 어떤 것을 1차 필수 마스터로 둘지 확정
- 예약 API URL, 요청값, 응답값 스펙 정리
