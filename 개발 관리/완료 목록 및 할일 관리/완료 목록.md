# 완료 목록

## 현재 판단

- 예약 기능 구현을 본격적으로 시작해도 되는 단계로 판단.
- 이유: 인증, 인가, 사용자/역할/권한 관리, 헤더 메뉴, 다국어, 테마, 기본 대시보드 진입, 사이트 설정 등 공통 기반이 먼저 갖춰져 있음.
- 현 상태는 "예약 기능을 올릴 플랫폼 기반 정리 완료, 예약 도메인 구현은 아직 시작 전"에 가까움.

## 완료된 기반 작업

### 공통 플랫폼 기반

- Next.js + Spring Boot + PostgreSQL 개발 환경 구성
- 프론트/백엔드 기본 프로젝트 구조 정리
- JWT 로그인/로그아웃/세션 복구 흐름 구성
- Spring Security 기반 인증/인가 구조 구성
- RBAC 기반 역할/권한 구조 구성
- 역할 CRUD, 권한 CRUD, 권한 카테고리 CRUD 구현
- 사용자 목록/역할 변경 관리 기능 구현
- 관리자 전용 접근 제어 구조 구성

### 운영/관리 화면 기반

- DB 기반 헤더 메뉴 관리 기능 구현
- 메뉴 노출 권한 연동 구조 구성
- 대시보드 진입 구조 및 기본 랜딩 화면 구성
- 사이트 설정 관리 API 및 화면 기반 구성
- 업로드용 S3 연동 기반 구성

### 사용자 경험 기반

- 다국어 i18n 구조 구성
- 언어 전환 UI 반영
- 브랜드 테마 전환 기능 구성
- 인증 화면과 기본 관리 화면 UI 구조 정리

## 아직 완료되지 않은 핵심 도메인

- 예약(Reservation/Booking) 도메인
- 고객(Customer) 도메인
- 직원/디자이너(Staff) 도메인
- 직원 스케줄/근무시간 도메인
- 시술 서비스/메뉴 도메인
- 예약 가능 시간 계산 로직
- 예약 생성/변경/취소 플로우
- 예약 상태 관리 및 운영 화면

## 메모

- 현재 대시보드는 예약 기능이 들어오기 전의 플레이스홀더 상태임.
- 즉, "무엇을 먼저 만들지"가 불명확한 단계는 지났고, 이제 예약 도메인 설계와 구현으로 넘어가면 됨.
