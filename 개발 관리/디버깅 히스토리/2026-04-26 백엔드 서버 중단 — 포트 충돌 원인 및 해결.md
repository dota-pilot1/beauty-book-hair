# 백엔드 서버 중단 — 포트 충돌 원인 및 해결

**날짜**: 2026-04-26  
**증상**: 운영 사이트 전체 502, 로그인 불가

---

## 원인

배포 스크립트에서 `pkill -f 'java -jar app.jar'` 실행 후 프로세스가 완전히 종료되기 전에 새 JAR가 기동을 시도해 포트 4101 충돌 발생.

```
Web server failed to start. Port 4101 was already in use.
```

`pkill`은 신호를 보낼 뿐 종료를 보장하지 않는다. `sleep 2`만으로는 부족한 경우가 있음.

---

## 해결

포트를 직접 점유 중인 PID를 찾아 강제 종료 후 재시작.

```bash
# 기존 방식 (불안정)
pkill -f 'java -jar app.jar'; sleep 2; ...

# 수정된 방식
lsof -ti:4101 | xargs kill -9 || true
sleep 2
set -a && source ~/.env && set +a
nohup java -jar app.jar --spring.profiles.active=prod > spring-boot.log 2>&1 &
```

---

## 재발 방지

- CLAUDE.md 배포 스크립트를 `lsof -ti:4101 | xargs kill -9` 방식으로 업데이트 완료
- 배포 후 `grep 'Started\|ERROR\|FAILED' spring-boot.log` 로 기동 성공 여부 확인
