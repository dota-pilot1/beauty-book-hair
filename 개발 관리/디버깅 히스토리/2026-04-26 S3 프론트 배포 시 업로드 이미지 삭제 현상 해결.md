# S3 프론트 배포 시 업로드 이미지 삭제 현상 해결

**날짜**: 2026-04-26  
**증상**: 서버 재배포 후 로그인 페이지 대문 이미지 및 기타 업로드 이미지가 모두 사라짐

---

## 원인

프론트엔드 정적 파일과 서버 업로드 이미지가 **같은 S3 버킷** (`beauty-book-hair-front`)을 사용 중.

| 용도 | 경로 |
|------|------|
| 프론트 정적 파일 | `s3://beauty-book-hair-front/` (루트) |
| 업로드 이미지 | `s3://beauty-book-hair-front/beauty-book/` (서브폴더) |

배포 시 `aws s3 sync out/ s3://beauty-book-hair-front --delete` 실행하면 `--delete` 옵션이 로컬 `out/`에 없는 파일(= `beauty-book/` 폴더 전체)을 삭제.

---

## 해결

`--exclude "beauty-book/*"` 옵션 추가로 업로드 이미지 폴더를 sync 대상에서 제외.

```bash
# 기존 방식 (이미지 삭제됨)
aws s3 sync out/ s3://beauty-book-hair-front --delete

# 수정된 방식
aws s3 sync out/ s3://beauty-book-hair-front --delete --exclude "beauty-book/*"
```

---

## 재발 방지

- CLAUDE.md 배포 가이드 업데이트 완료
- 이미 삭제된 이미지는 어드민 `/site-settings` 에서 재업로드 필요
- 근본 해결책: 추후 이미지 전용 버킷 분리 고려 (`beauty-book-hair-uploads` 등)
