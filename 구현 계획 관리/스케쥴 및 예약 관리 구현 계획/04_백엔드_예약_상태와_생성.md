# 04. 백엔드 예약 상태와 생성

## 목표

예약 테이블과 기본 예약 생성 API를 먼저 만든다.

이 단계에서 관리자 화면을 완성하지 않아도 된다. 하지만 슬롯 계산에서 기존 예약을 제외하려면 예약 데이터 모델은 필요하다.

## 구현 파일 범위

```text
beauty-book-server/src/main/java/com/cj/beautybook/reservation/
```

## 엔티티

### Reservation

필드:

- id
- customerId
- customerName
- customerPhone
- designerId
- beautyServiceId
- startAt
- endAt
- status
- customerMemo
- adminMemo
- createdAt
- updatedAt

### ReservationStatus

- REQUESTED
- CONFIRMED
- CANCELLED_BY_CUSTOMER
- CANCELLED_BY_ADMIN
- COMPLETED
- NO_SHOW

## Service

### ReservationService

역할:

- 예약 생성
- 예약 단건 조회
- 고객 전화번호 기준 예약 조회
- 관리자 예약 목록 조회
- 상태 변경
- 시간 중복 검증

## 예약 생성 흐름

1. 요청에서 designerId, beautyServiceId, startAt을 받는다.
2. 선택한 디자이너가 해당 시술을 수행할 수 있는지 검증한다.
3. 시술의 durationMinutes를 조회한다.
4. endAt을 계산한다.
5. reservation-slots 계산 로직으로 해당 시간이 AVAILABLE인지 검증한다.
6. 기존 REQUESTED/CONFIRMED 예약과 겹치는지 재검증한다.
7. Reservation을 REQUESTED 상태로 저장한다.

## 상태 변경 흐름

- REQUESTED -> CONFIRMED
- REQUESTED -> CANCELLED_BY_ADMIN
- CONFIRMED -> CANCELLED_BY_ADMIN
- CONFIRMED -> COMPLETED
- CONFIRMED -> NO_SHOW

고객 취소는 별도 API로 둔다.

## 완료 기준

- 고객 예약 요청을 저장할 수 있다.
- 시술을 수행할 수 없는 디자이너로 예약 요청하면 실패한다.
- 같은 디자이너의 같은 시간 중복 예약을 막는다.
- 취소된 예약은 시간 점유에서 제외된다.
