# 03. 백엔드 스케쥴 기본 관리

## 목표

관리자가 영업시간, 디자이너 근무시간, 예약 불가 시간을 관리할 수 있게 한다.

## 구현 파일 범위

```text
beauty-book-server/src/main/java/com/cj/beautybook/schedule/
```

## 엔티티

- BusinessHour
- DesignerWorkingHour
- BlockedTime
- BlockType

## Repository

- BusinessHourRepository
- DesignerWorkingHourRepository
- BlockedTimeRepository

필요 조회:

- 요일별 매장 영업시간 조회
- 디자이너 + 요일 근무시간 조회
- 특정 날짜 구간에 걸치는 차단 시간 조회
- 매장 전체 차단 시간 조회

## Service

### ScheduleService

역할:

- 영업시간 목록 조회/수정
- 디자이너 근무시간 목록 조회/수정
- 예약 불가 시간 생성/수정/삭제

## Controller

관리자 API로 둔다.

- GET /api/admin/schedules/business-hours
- PUT /api/admin/schedules/business-hours
- GET /api/admin/schedules/designers/{designerId}/working-hours
- PUT /api/admin/schedules/designers/{designerId}/working-hours
- GET /api/admin/schedules/blocked-times
- POST /api/admin/schedules/blocked-times
- PATCH /api/admin/schedules/blocked-times/{blockedTimeId}
- DELETE /api/admin/schedules/blocked-times/{blockedTimeId}

## 검증

- 시작 시간이 종료 시간보다 빨라야 한다.
- 영업시간 범위가 비어 있으면 안 된다.
- 같은 디자이너의 차단 시간이 겹치는 경우 허용 여부를 정한다.
- MVP에서는 겹침 허용보다 거절이 낫다.

## 완료 기준

- 관리자 API로 영업시간을 저장/조회할 수 있다.
- 디자이너별 근무시간을 저장/조회할 수 있다.
- 특정 날짜에 예약 불가 시간을 추가하면 슬롯 계산에서 제외할 준비가 된다.
- 디자이너 근무시간은 디자이너가 특정 시술을 수행할 수 있는지와 별개로 관리된다.
