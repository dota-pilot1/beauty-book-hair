# 11. 검증 테스트 시나리오

## 백엔드 단위 테스트

### 슬롯 계산

- 영업시간이 닫힌 요일이면 BLOCKED 슬롯 또는 빈 응답 정책이 적용된다.
- 선택한 디자이너가 선택한 시술을 수행할 수 없으면 슬롯 조회가 실패한다.
- 디자이너가 쉬는 요일이면 BLOCKED 슬롯 또는 빈 응답 정책이 적용된다.
- 30분 단위로 후보 시작 시간이 생성된다.
- 영업시간과 근무시간의 교집합 밖은 BLOCKED 상태가 된다.
- 차단 시간과 겹치는 슬롯은 BLOCKED 상태가 된다.
- 확정 예약과 겹치는 슬롯은 RESERVED 상태가 된다.
- 승인 대기 예약과 겹치는 슬롯은 REQUESTED 상태가 된다.
- 시술 소요시간이 들어갈 수 없는 마지막 시간 제외
- 취소된 예약은 슬롯 차단하지 않음

### 예약 생성

- 가능한 슬롯이면 REQUESTED 예약 생성
- 불가능한 슬롯이면 실패
- 이미 예약된 시간과 겹치면 실패
- 없는 시술이면 실패
- 없는 디자이너면 실패
- 해당 시술을 수행할 수 없는 디자이너면 실패

### 상태 변경

- REQUESTED -> CONFIRMED 가능
- CONFIRMED -> COMPLETED 가능
- CANCELLED -> CONFIRMED 불가
- 존재하지 않는 예약 상태 변경 실패

## 프론트 검증

### 고객 예약 화면

- 시술 목록이 API 데이터로 보인다.
- 시술 이미지가 있으면 카드에 보인다.
- 시술 선택 후 다음 단계로 이동 가능하다.
- 시술 선택 후 해당 시술을 수행 가능한 디자이너만 보인다.
- 디자이너 선택 후 날짜/시간 슬롯이 조회된다.
- AVAILABLE 슬롯만 클릭 가능하다.
- REQUESTED, RESERVED, BLOCKED 슬롯은 disabled로 보인다.
- 예약 요청 성공 시 요약이 유지된다.
- 예약 요청 실패 시 슬롯을 다시 조회한다.

### 관리자 스케쥴 화면

- 영업시간 수정 후 저장된다.
- 디자이너 근무시간 수정 후 저장된다.
- 차단 시간 추가/삭제가 가능하다.
- 차단 시간 추가 후 고객 슬롯이 BLOCKED 상태로 보인다.

### 관리자 예약 화면

- 예약 목록 필터가 동작한다.
- 예약 상세를 볼 수 있다.
- 상태 변경 후 목록에 반영된다.

## 수동 QA 핵심 흐름

1. 관리자가 월요일 영업시간을 10:00-20:00으로 저장한다.
2. 디자이너 A 근무시간을 11:00-18:00으로 저장한다.
3. 13:00-14:00 차단 시간을 추가한다.
4. 고객 화면에서 60분 시술을 선택한다.
5. 해당 시술을 수행 가능한 디자이너만 보이는지 확인한다.
6. 디자이너 A를 선택한다.
7. 13:00, 13:30 슬롯이 BLOCKED로 보이는지 확인한다.
8. 11:00 예약을 요청한다.
9. 같은 시간과 겹치는 슬롯이 REQUESTED로 바뀌는지 확인한다.
10. 관리자 예약 목록에서 REQUESTED 예약을 CONFIRMED로 바꾼다.
11. 내 예약 화면에서 확정 상태가 보이는지 확인한다.
