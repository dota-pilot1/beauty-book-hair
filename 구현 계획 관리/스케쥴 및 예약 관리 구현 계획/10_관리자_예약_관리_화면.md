# 10. 관리자 예약 관리 화면

## 목표

관리자가 예약 요청을 보고 상태를 변경할 수 있게 한다.

## 페이지

```text
beauty-book--front/src/app/reservations/page.tsx
```

또는 관리 메뉴 기준으로:

```text
beauty-book--front/src/app/admin/reservations/page.tsx
```

## 화면 구성

### 필터

- 날짜
- 상태
- 디자이너
- 고객명/전화번호 검색

### 목록

처음에는 테이블형이 적합하다.

컬럼:

- 예약 시간
- 고객명
- 연락처
- 시술
- 디자이너
- 상태
- 요청사항
- 액션

### 상세

예약 행 클릭 시 다이얼로그.

- 고객 정보
- 시술 정보
- 예약 시간
- 상태 변경 이력
- 고객 요청사항
- 관리자 메모

## 상태 액션

REQUESTED:

- 확정
- 관리자 취소

CONFIRMED:

- 완료
- 노쇼
- 관리자 취소

CANCELLED:

- 읽기 전용

## 캘린더 보기

2차 개선으로 둔다.

초기에는 목록/테이블이 낫다. 상태 관리, 검색, 필터가 예약 운영에는 더 중요하다.

## 완료 기준

- 고객이 예약 요청한 데이터가 목록에 나온다.
- 관리자가 확정/취소/완료/노쇼 처리할 수 있다.
- 상태 변경 후 고객 내 예약 화면에 반영된다.
