# 03-1. 백엔드 디자이너 시술 매핑

## 목표

고객이 시술을 먼저 선택하면, 해당 시술을 수행할 수 있는 디자이너만 보여준다.

## 구현 파일 범위

```text
beauty-book-server/src/main/java/com/cj/beautybook/designer/
```

## 엔티티

### Designer

- id
- name
- profileImageUrl
- introduction
- active
- displayOrder

### DesignerService

- id
- designerId
- beautyServiceId
- active

## Repository

- DesignerRepository
- DesignerServiceRepository

필요 조회:

- active 디자이너 목록
- beautyServiceId 기준 가능한 디자이너 목록
- designerId + beautyServiceId 가능 여부

## Service

### DesignerService

역할:

- 시술 가능한 디자이너 목록 조회
- 디자이너가 특정 시술을 수행할 수 있는지 검증
- 관리자용 디자이너-시술 매핑 관리

## 공개 API

```http
GET /api/designers?beautyServiceId=10
```

응답:

```json
[
  {
    "id": 1,
    "name": "수아 디자이너",
    "profileImageUrl": "https://example.com/designer/sua.jpg",
    "introduction": "레이어드 컷과 자연스러운 펌을 잘합니다."
  }
]
```

## 관리자 API

```http
GET /api/admin/designers
POST /api/admin/designers
PATCH /api/admin/designers/{designerId}
GET /api/admin/designers/{designerId}/services
PUT /api/admin/designers/{designerId}/services
```

## 예약 검증 연동

예약 생성 시 반드시 아래를 검증한다.

```text
선택한 designerId가
선택한 beautyServiceId를 수행 가능한가?
```

가능하지 않으면 예약 요청을 거절한다.

## 완료 기준

- 시술 선택 후 가능한 디자이너만 조회할 수 있다.
- 예약 생성 시 디자이너-시술 가능 여부를 서버에서 재검증한다.
- 관리자 화면에서 디자이너별 가능한 시술을 관리할 준비가 된다.
