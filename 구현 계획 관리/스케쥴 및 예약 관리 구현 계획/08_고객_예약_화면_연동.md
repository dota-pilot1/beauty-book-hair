# 08. 고객 예약 화면 연동

## 목표

현재 고객 예약 화면의 하드코딩 시간 슬롯을 제거하고 API 데이터로 연결한다.

## 현재 교체 대상

```text
beauty-book--front/src/app/booking/page.tsx
beauty-book--front/src/features/booking/model/bookingFlowStore.ts
```

## 단계

### 1단계: 선택 상태 정리

문자열 대신 ID/ISO 시간 중심으로 상태를 둔다.

- selectedServiceId
- selectedDesignerId
- selectedStartAt
- selectedEndAt

### 2단계: 서비스 선택

이미 구현된 시술 목록 API 훅을 사용한다.

- visible=true 시술만 노출
- imageUrls 첫 번째 이미지를 카드 대표 이미지로 사용

### 3단계: 가능한 시간 패널

시술을 선택하면 오른쪽 패널에서 날짜별 가능한 시간대를 바로 보여준다.

조회:

- `useReservationSlots(selectedServiceId, date)`
- 확장 시 `useReservationSlotDays(selectedServiceId, fromDate, days)`

저장:

- selectedStartAt
- selectedEndAt
- selectedSlotAvailableDesigners

정책:

- 디자이너를 먼저 선택하지 않아도 시술 + 날짜 기준으로 가능한 시간 전체를 보여준다.
- 각 시간 카드에는 가능한 디자이너 이름을 함께 표시한다.
- 가능한 디자이너가 없으면 해당 시간 카드는 disabled 처리한다.
- 시간 카드를 선택하면 선택한 시술 duration에 맞춰 차지하는 시간 범위를 즉시 강조한다.
- 선택한 시간에 가능한 디자이너가 1명뿐이면 자동 선택한다.
- 선택한 시간에 가능한 디자이너가 여러 명이면 카드 안 또는 카드 아래에서 디자이너를 선택하게 한다.
- 선호 디자이너 선택은 필수 단계가 아니라 optional 필터로 둔다.

### 4단계: 디자이너 선택 또는 확정

시간을 먼저 고른 뒤 해당 시간에 가능한 디자이너 중 한 명을 선택한다.

조회:

- 시간 카드의 `availableDesigners`

저장:

- selectedDesignerId

정책:

- 시술을 바꾸면 기존 selectedDesignerId, selectedStartAt, selectedEndAt을 초기화한다.
- 날짜나 시간 카드를 바꾸면 기존 selectedDesignerId를 재검증한다.
- 가능한 디자이너가 1명뿐이면 자동 선택 가능하다.
- 선호 디자이너 필터가 있으면 reservation-slots 요청에 optional `designerId`를 붙인다.

### 5단계: 날짜/시간 선택 상세

슬롯은 가능한 시간만 보여주지 않는다. 전체 시간대를 30분 단위 후보로 보여주고 상태에 따라 클릭 가능 여부를 나눈다.

슬롯 계산 기준:

- 선택한 시술의 durationMinutes
- 선택한 시술을 수행 가능한 디자이너 목록
- 각 디자이너의 근무시간
- 각 디자이너의 차단 시간
- 각 디자이너의 REQUESTED/CONFIRMED 예약
- 매장 전체 영업시간과 차단 시간

상태:

- AVAILABLE: 예약 요청 가능
- REQUESTED: 승인 대기 예약과 겹침
- RESERVED: 확정 예약과 겹침
- BLOCKED: 영업시간 외, 근무시간 외, 차단 시간

선택 가능:

- AVAILABLE만 가능
- REQUESTED, RESERVED, BLOCKED는 disabled
- AVAILABLE 카드 선택 시 `occupiedUnitCount`만큼 연결된 범위를 하이라이트
- 하이라이트는 UX 표시이며 최종 가능 여부는 서버 응답과 예약 생성 재검증을 따른다.

빈 상태:

- 시술 선택 필요
- 선택한 날짜에 가능한 시간이 없음
- 예약 가능 시간이 없음

로딩 상태:

- 슬롯 카드 skeleton

### 6단계: 예약 요청

확인 단계에서 `useCreateReservation`으로 POST한다.

성공 후:

- 예약 요청 완료 화면
- 내 예약 페이지로 이동 버튼

실패 후:

- 이미 예약된 시간 안내
- 슬롯 재조회

## UI 방향

- 서비스 카드는 이미지 포함 카드형 유지
- 예약 요약은 하단 카드 유지
- 오른쪽은 오늘 예약 현황보다 “날짜별 가능 시간” 영역을 우선한다.
- 시간 카드는 `시간 범위 + 가능한 디자이너 이름 + 가능 인원`을 포함한다.
- 선택된 시간 카드는 시술 소요시간만큼 차지하는 범위를 연결된 블록처럼 보여준다.
- 선호 디자이너는 필터 UI로 제공하고 필수 단계처럼 보이지 않게 한다.
- 공간이 부족하면 오른쪽 예약 요약을 다시 올리지 않는다.
- 시간 표기는 `오전 10:30 ~ 오전 11:30`처럼 시작~종료 범위로 보여준다.

## 완료 기준

- 시술 목록은 DB 등록 데이터와 연동된다.
- 시간 슬롯은 시술 + 날짜 기준 reservation-slots API에서 온다.
- 각 시간 카드에는 가능한 디자이너 이름이 함께 보인다.
- 시간 선택 후 가능한 디자이너 중 한 명을 선택하거나 자동 선택된다.
- 선택한 시간은 시술 소요시간만큼 차지하는 범위가 즉시 표시된다.
- 예약됨/요청 중/불가 슬롯은 disabled로 보인다.
- 예약 요청을 보내면 관리자 예약 목록에 나타난다.
