# 이메일 서비스 관련 구현 내역 및 파일 구조

## 구현 개요

BeautyBook 이메일 서비스는 Spring Boot 서버의 `NotificationService` 인터페이스와 `EmailNotificationService` 구현체를 통해 발송된다.

현재 Google SMTP는 발송 수단이고, 실제 수신자는 서비스 로직에서 지정한다. 예약 요청 메일은 고객에게 예약 접수 안내를 보내고, 관리자가 설정한 예약 요청 수신 이메일이 있으면 관리자에게도 별도 알림 메일을 보낸다.

## 현재 발송 대상

| 이벤트 | 수신자 | 설명 |
| --- | --- | --- |
| 회원가입 | 고객 이메일 | 가입 환영 메일 |
| 비밀번호 재설정 | 고객 이메일 | 재설정 링크 안내 |
| 비밀번호 변경 | 고객 이메일 | 변경 완료 및 보안 안내 |
| 예약 요청 | 고객 이메일 | 예약 접수 완료 안내 |
| 예약 요청 | 관리자 설정 이메일 | 새 예약 요청 확인 안내 |
| 예약 확정 | 고객 이메일 | 예약 확정 안내 |
| 고객 예약 취소 | 고객 이메일 | 고객 취소 완료 안내 |
| 관리자 예약 취소 | 고객 이메일 | 매장 취소 안내 및 사유 |

## 예약 요청 관리자 알림

예약 요청은 관리자가 승인해야 하는 흐름이므로 관리자에게도 알림이 필요하다.

처리 흐름:

1. 고객이 예약 요청을 생성한다.
2. `ReservationService.create(...)`에서 예약을 저장한다.
3. `notificationService.sendReservationRequested(saved)`를 호출한다.
4. `EmailNotificationService.sendReservationRequested(...)`가 고객에게 접수 메일을 보낸다.
5. `site_settings.reservation_request_email` 값이 있으면 해당 이메일로 관리자 알림 메일을 보낸다.

관리자 이메일이 비어 있으면 기존처럼 고객에게만 예약 접수 메일이 발송된다.

## 설정 저장 방식

별도 메일 설정 테이블을 새로 만들지 않고 기존 `site_settings` 싱글톤 테이블을 확장했다.

추가 컬럼:

```text
reservation_request_email
```

선택 이유:

- 사이트 전역 설정이 이미 `site_settings` 한 row로 관리되고 있다.
- 예약 요청 수신 이메일도 전역 운영 설정에 가깝다.
- 현재 요구사항은 이메일 한 개라 별도 테이블보다 단순한 컬럼 추가가 적합하다.

## API 설계

기존 `/api/site-settings`는 공개 조회 API이므로 관리자 이메일을 포함하지 않는다.

메일 설정은 관리자 전용 API로 분리했다.

| Method | Path | 권한 | 설명 |
| --- | --- | --- | --- |
| GET | `/api/site-settings` | 공개 | 대문 이미지, 소개 문구 조회 |
| PUT | `/api/site-settings` | `ROLE_ADMIN` | 대문 이미지, 소개 문구 수정 |
| GET | `/api/site-settings/mail` | `ROLE_ADMIN` | 예약 요청 수신 이메일 조회 |
| PUT | `/api/site-settings/mail` | `ROLE_ADMIN` | 예약 요청 수신 이메일 수정 |

공개 사이트 설정 응답에 관리자 이메일을 넣지 않은 이유:

- 로그인/회원가입 화면 등 공개 화면에서도 `/api/site-settings`를 사용한다.
- 관리자 이메일이 외부에 노출될 수 있다.
- 메일 설정은 운영 정보라 관리자 전용 API로 분리하는 것이 안전하다.

## 관리자 화면

관리 페이지는 `/mail-settings`로 추가했다.

메뉴명은 `메일 관리`로 정리했다.

이름 판단 기준:

- `사이트 관리`는 화면, 소개, 브랜드, 기본 정보까지 넓게 느껴진다.
- 현재 기능은 예약 요청 알림 메일 수신처 설정에 집중되어 있다.
- 관리자가 찾기에는 `메일 관리`가 가장 직접적이다.
- 추후 SMS, 카카오톡, 푸시까지 확장되면 `알림 관리`로 변경하는 것이 적합하다.

현재 관리 메뉴 구조:

```text
화면 관리: 대문 이미지, 소개 문구
메일 관리: 예약 요청 수신 이메일
```

## 백엔드 파일 구조

```text
beauty-book-server/src/main/java/com/cj/beautybook/notification/application/
├── NotificationService.java
└── EmailNotificationService.java
```

역할:

- `NotificationService.java`
  - 이메일 알림 기능의 인터페이스
  - 회원가입, 비밀번호, 예약 상태 변경 메서드 정의

- `EmailNotificationService.java`
  - `JavaMailSender`를 사용해 HTML 메일 발송
  - 예약 요청 시 고객 메일과 관리자 알림 메일 발송
  - 관리자 수신 이메일은 `SiteSettingRepository`에서 조회

```text
beauty-book-server/src/main/java/com/cj/beautybook/site_settings/
├── domain/
│   └── SiteSetting.java
├── application/
│   └── SiteSettingService.java
├── infrastructure/
│   └── SiteSettingRepository.java
└── presentation/
    ├── SiteSettingController.java
    └── dto/
        ├── SiteSettingResponse.java
        ├── UpdateSiteSettingRequest.java
        ├── MailSettingResponse.java
        └── UpdateMailSettingRequest.java
```

역할:

- `SiteSetting.java`
  - 기존 사이트 설정 엔티티
  - `reservationRequestEmail` 필드 추가
  - 빈 문자열은 `null`로 정규화

- `SiteSettingService.java`
  - 공개 사이트 설정 조회/수정
  - 관리자 전용 메일 설정 조회/수정

- `SiteSettingController.java`
  - `/api/site-settings`
  - `/api/site-settings/mail`

- `MailSettingResponse.java`
  - 관리자 전용 메일 설정 응답

- `UpdateMailSettingRequest.java`
  - 관리자 전용 메일 설정 수정 요청
  - `@Email`, `@Size(max = 255)` 검증

```text
beauty-book-server/src/main/java/com/cj/beautybook/reservation/application/
└── ReservationService.java
```

역할:

- 예약 생성 후 `notificationService.sendReservationRequested(saved)` 호출
- 메일 발송 실패 시 예약 생성 자체는 실패시키지 않고 warning 로그만 남김

```text
beauty-book-server/src/main/java/com/cj/beautybook/config/
└── MenuSeeder.java
```

역할:

- 관리자 드롭다운 메뉴에 `메일 관리` 항목 seed 추가
- 경로: `/mail-settings`

## 운영 시 반영 조건

프론트 화면만 변경된 경우 브라우저 새로고침으로 확인 가능하다.

이번 변경은 백엔드 API와 DB 컬럼이 추가되었으므로 백엔드 재시작이 필요하다.

현재 서버 설정:

```yaml
spring:
  jpa:
    hibernate:
      ddl-auto: update
```

따라서 백엔드 재시작 시 `site_settings` 테이블에 `reservation_request_email` 컬럼이 자동 반영된다.

## 검증 내역

서버 테스트:

```bash
cd beauty-book-server
./gradlew test
```

결과:

```text
BUILD SUCCESSFUL
```

프론트 빌드:

```bash
cd beauty-book--front
npm run build
```

결과:

```text
Compiled successfully
```

참고:

```bash
npm run lint
```

프론트 lint는 기존 작업트리의 다른 파일들에서 실패 중이다. 이번 메일 설정 변경과 직접 관련 없는 `react-hooks/set-state-in-effect`, `no-explicit-any` 등의 기존 오류가 포함되어 있다.
