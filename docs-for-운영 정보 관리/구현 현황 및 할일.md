## 알려진 이슈 및 주의사항

### 운영 DB 초기 세팅 주의
- `business_hours` 테이블 초기값에서 SUNDAY가 `closed = true`로 들어가 있을 수 있음
- 배포 후 반드시 `/schedule` 어드민 페이지에서 확인/수정 필요
- `staff_working_hours`도 초기값에서 SUNDAY `working = false` 가능성 있음 → `/staff` 스케쥴 모달로 수정

### 운영 배포 후 체크리스트
```
□ /schedule 접속 → 요일별 영업시간 확인 (특히 SUNDAY)
□ /staff 접속 → 직원별 스케쥴 모달 → SUNDAY 근무 여부 확인
□ /booking 접속 → 오늘 날짜 슬롯 정상 노출 확인
```

### logback ThrowableProxy (운영 환경)
- **증상**: 클라이언트 소켓 끊길 때 HTTP 스레드 사망 → 응답 전달 안 됨
- **원인**: Tomcat 클래스로더 ↔ Spring Boot 중첩 클래스로더 분리 문제
- **해결**: `application.yaml`에 Tomcat 로거 OFF 적용 완료 (`org.apache.tomcat/catalina/coyote: OFF`)
- 상세: `개발 관리/디버깅 히스토리/2026-04-26 운영 예약 요청 응답 없음 (logback ThrowableProxy).md`

## 배포 명령어 요약

### 백엔드 배포 (EC2)
```bash
# 1. 빌드
cd beauty-book-server
./gradlew clean build -x test

# 2. 업로드
scp -i "배포 가이드/hibot-d-server-key.pem" \
  build/libs/beauty-book-server-*.jar \
  ubuntu@13.124.117.243:~/app.jar

# 3. 재시작
ssh -i "배포 가이드/hibot-d-server-key.pem" ubuntu@13.124.117.243 \
  "pkill -f 'java -jar app.jar'; sleep 2; set -a && source ~/.env && set +a; nohup java -jar app.jar --spring.profiles.active=prod > spring-boot.log 2>&1 &"
```

### 프론트엔드 배포 (S3 + CloudFront)
```bash
# 1. 빌드 및 S3 업로드
cd beauty-book--front
npm run build
aws s3 sync out/ s3://beauty-book-hair-front --delete

# 2. CloudFront 캐시 무효화
aws cloudfront create-invalidation \
  --distribution-id E11NF3HMOB52NI \
  --paths "/*"
```
