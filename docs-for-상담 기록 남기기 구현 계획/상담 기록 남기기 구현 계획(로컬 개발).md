# 상담 기록 남기기 - 로컬 개발 구현 계획

## 현재 상태

백엔드에 `voice` 도메인이 이미 구현되어 있음:
- `Call` 엔티티 (callSid, fromNumber, toNumber, status, startedAt, endedAt, durationSec)
- `GET /api/voice` — 상담 목록 조회
- `POST /api/voice/inbound` — Twilio 웹훅 전용 (로컬에서 직접 호출 불가)
- `POST /api/voice/status` — Twilio 상태 콜백 전용

**문제**: 기존 엔드포인트는 Twilio가 호출하는 구조라 로컬에서 직접 테스트 불가

---

## 로컬 개발 전략

Twilio 없이 **프론트에서 버튼으로 통화 시작/종료를 시뮬레이션**하는 전용 API 추가

---

## 구현 순서

### Step 1. 백엔드 — 로컬 시뮬레이션 API 추가

`POST /api/calls/start` — 통화 시작 (callSid 자동 생성)
```json
{ "fromNumber": "010-1234-5678", "toNumber": "02-0000-0000" }
```

`PATCH /api/calls/{callSid}/end` — 통화 종료
```json
{ "durationSec": 120 }
```

`GET /api/calls` — 상담 목록 (최신순)

> 기존 `/api/voice` 엔드포인트는 건드리지 않고 별도 컨트롤러로 추가

---

### Step 2. 백엔드 — 응답 DTO 추가

`Call` 엔티티 직접 반환 → `CallResponse` DTO로 교체
- id, callSid, fromNumber, toNumber, status, startedAt, endedAt, durationSec

---

### Step 3. 프론트 — 대시보드 UI 구현

현재 대시보드는 placeholder만 있음. 아래 화면 구성:

```
[ 통화 시작 ] 버튼
  → fromNumber, toNumber 입력 후 POST /api/calls/start 호출

상담 목록 테이블
  | 발신번호 | 수신번호 | 시작시간 | 종료시간 | 통화시간 | 상태 | 액션 |
  → 진행 중인 통화에만 [ 종료 ] 버튼 표시
  → [ 종료 ] 클릭 시 PATCH /api/calls/{callSid}/end 호출
```

---

### Step 4. 프론트 — API 연동

```
entities/call/
  model/types.ts         — Call 타입 정의
  api/callApi.ts         — startCall, endCall, getCalls

features/call/
  CallStartForm.tsx      — 통화 시작 폼
  CallList.tsx           — 상담 목록 테이블

app/dashboard/page.tsx   — 위 컴포넌트 조합
```

---

## 완성 후 흐름

```
1. 로그인
2. 대시보드에서 [ 통화 시작 ] 클릭 → fromNumber/toNumber 입력
3. POST /api/calls/start → DB에 RINGING 상태로 저장
4. 목록에 진행 중 통화 표시
5. [ 종료 ] 클릭 → PATCH /api/calls/{callSid}/end
6. DB 업데이트 (COMPLETED, endedAt, durationSec)
7. 목록 갱신
```

---

## 나중에 실제 Twilio 연동 시

로컬 시뮬레이션 API를 그대로 두고, Twilio 웹훅(`/api/voice/inbound`, `/api/voice/status`)이 동일한 `VoiceService` 로직을 호출하도록 연결만 하면 됨. 구조 변경 없음.
